  1. Should I go directly to grpc?

Going to grpc will complicate the deployment. As a one man shop I don’t need that complexity. The added benefit for a todo like app is not much, the backend is simple enough, does not need a cluster of micro services.

  1. Should I use symbol link or dart package to allow the protobuf code sharing? Symbolic link does not work reliabliy on all platforms. Instead if I publis the definition to a dart lib, it will work on all platforms. Because the dart team will spend time maintaining it and make sure it does not break.

The command used to use when need to import message definition in more than one files:

protoc --dart_out=./dart -I ./proto ./proto/*.proto

Should I pick firebase or supabase?

Supabase can ba self hosted: https://www.youtube.com/watch?v=FqiQKRKsfZE

Basic requirement that can be fulfilled by both:

  1. anonymized user login
  2. read and write of data
  3. social login such as fb/google

The drawback of using supabase:

  1. missing wechat as the oauth provider. However if we want to add it, we can learn it here: https://github.com/orgs/supabase/discussions/417
  2. And something about the jwt token: https://medium.com/@gracew/using-supabase-rls-with-a-custom-auth-provider-b31564172d5d
  3. Also some current hack: https://github.com/orgs/supabase/discussions/6547#discussioncomment-9049664

Overall, the Supabase allows me to mirgate to my self-host instance when my project grows large enough. There is problem to integarte with WeChat auth but that can be solved by contributing to the auth repo, someone may do it before me, or I can do it myself.
